Purpose

STMCube is an STMicroelectronics original initiative to ease developers life by reducing development efforts, time and cost.

STM32Cube covers STM32 portfolio.

STM32Cube Version 1.x includes:

  • The STM32CubeMX, a graphical software configuration tool that allows to generate C initialization code using graphical wizards.

  • A comprehensive embedded software platform, delivered per series

    • The STM32Cube HAL, an STM32 abstraction layer embedded software, ensuring maximized portability across STM32 portfolio.HAL APIs are available for all peripherals.

    • Low-layer APIs (LL) offering a fast light-weight expert-oriented layer which is closer to the hardware than the HAL. LL APIs are available only for a set of peripherals.

    • A consistent set of middleware components such as FreeRTOS, OpenAMP.

    • All embedded software utilities, delivered with a full set of examples.

The STM32Cube firmware solution offers a straightforward API with a modular architecture, making it simple to fine tune custom applications and scalable to fit most requirements.

The drivers provided within this package support the STM32MP251/253/257 lines.

Notes

  • All the examples/projects delivered in this release are meant to be run in Production mode.
  • Production boot mode
    • Firmware is loaded in coprocessor by the master processor (Cortex A35 - Flash Memory boot mode)
    • You would need to install the OSTL images on SDCard (please refer to the ST Wiki)
    • OSTL images come with different configurations and images from each configuration can be flashed on a storage media like SDCard.
      • Path of configuration files (.tsv files): â€˜images/stm32mp2/flashlayout_st-image-weston/optee/â€™
    • The configuration suitable to run the above examples is the one with ca35tdcid-ostl-m33-examples embedded in the name.
  • ResourceManager is used in all examples to request/acquire hardware resources needed for the functionality demonstrated.
  • To build examples/apps using STM32CubeIDE:
    • Select config CA35TDCID_m33_ns_sign :
    • Building using selected config generates a signed firmware binary.
  • Copy signed firmware binaries to the root filesystem on the board (Assuming you have connected to board on USB Port).
    $ ssh root@192.168.7.1  'mkdir -p /home/root/<EXAMPLE_NAME>/lib/firmware'
    $ scp <SIGNED_FIRMWARE> root@192.168.7.1:/home/root/<EXAMPLE_NAME>/lib/firmware/
    $ scp fw_cortex_m33.sh root@192.168.7.1:/home/root/<EXAMPLE_NAME>/
  • To run examples/apps on target:
    • On the target shell run: 'cd /home/root/<EXAMPLE_NAME>/ ; ./fw_cortex_m33.sh start'

Demonstrations

  • USBPD_DRP_UCSI: This demonstration project would be functional when OSTL images are flashed with configuration files (.TSV file) (without ca35tdcid-ostl-m33-examples embedded in the name.).

LTDC 3rd Layer Example

  • This example demonstrates how we can display some content on Layer 3 of LTDC from CM33 while Layer 1 and 2 are being used by CA35.
  • How to Run:
    • Compile the example and copy it to a folder on the root filesystem on the board as described above.
    • Use the configuration ca35tdcid-ostl-m33-examples while flashing the OSTL images on SDcard.
    • Connect the panel to the EV1 board.
    • When you boot the system, the default animation and display controlled by CA35(running Linux) will show up rendering on Layer 1 and Layer 2.
    • Now start the LTDC example, It will display the contents on Layer3 while the content from CA35 will still be visible in the back layers.
